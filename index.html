Selected work

AI-native prototypes for expression, focus, and clarity

Journal Scrapbook Buddy

AI Expression · Journaling UX · JavaScript

A playful AI-inspired journaling prototype where users write what they are carrying, select multiple colors and a vibe, generate a personalized scrapbook page, save scraps to a growing local book, and export the current page as a PDF. I built it to explore how AI can support reflection and self-expression without making the experience feel clinical.

Live demo GitHub repo Design process

Design signal

  • Designed a repeatable journaling system with emotional input, multi-color customization, AI-style reflection, save-to-book, and PDF export.
  • Documented product framing, low-fidelity wireframes, interaction states, content design decisions, prompt strategy, and accessibility notes.
  • Created a cute visual language that makes reflection feel approachable while keeping the user in control.

Responsible AI framing

  • Frames outputs as creative reflection, not therapy, diagnosis, or life advice.
  • Uses local browser storage for saved scraps in the prototype.
  • Models a future secure AI architecture where journal content would be handled through a backend, not exposed client-side.

Focus Bakery

AI Focus Companion · Gamified UX · JavaScript

A bakery-themed focus companion where each completed focus round unlocks an ingredient. After five rounds, the user bakes a cute treat and adds it to a growing shop collection. The prototype reframes productivity as a warm progression system with AI-style reflection instead of a pressure-based task tracker.

Live demo Design process

Interaction design

  • Designed a clear loop: focus round → ingredient → recipe → baked item → shop collection.
  • Created a scalable recipe system with multiple bakery items, progress states, collection cards, and local persistence.
  • Added AI-style bakery notes so users can turn completed work into warm, reviewable reflection.

Meta alignment

  • Shows systems thinking, functional prototyping, UX writing, visual design, and front-end implementation.
  • Uses playful interaction patterns to make productivity feel motivating without relying on guilt or pressure.
  • Documents wireframes, content decisions, accessibility considerations, and future AI integration paths.

Twitter Storage & Retrieval System

Python · MySQL · Redis · SQL

A Twitter-style backend system built to support tweet posting and home timeline retrieval at scale. I designed relational schemas for users, tweets, and follower relationships, implemented API-style functions like postTweet and getHomeTimeline, and benchmarked relational versus in-memory storage tradeoffs.

Systems thinking

  • Benchmarked 1M tweet insertions and 50K timeline requests.
  • Improved insertion throughput from approximately 3,906 to 5,574 calls per second using Redis.
  • Evaluated read/write performance tradeoffs for high-volume social media workloads.
