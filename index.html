Journal Scrapbook Buddy

Journaling UX · AI-Inspired Reflection · JavaScript

A playful journaling prototype where users write what they are carrying, select multiple colors and a vibe, generate a personalized scrapbook page, save scraps to a growing local book, and export the current page as a PDF. The project explores how reflection tools can feel creative, gentle, and personal instead of blank or clinical.

Live demo GitHub repo Design process A/B test

What I built

  • Designed a repeatable journaling system with emotional input, multi-color customization, generated reflection, save-to-book, and PDF export.
  • Used browser localStorage to preserve saved scraps in a growing personal book.
  • Created a soft visual language that makes reflection feel approachable while keeping the user in control.

Process

  • Documented product framing, low-fidelity wireframes, interaction states, content decisions, prompt strategy, and accessibility notes.
  • Defined boundaries around generated reflection so outputs stay creative and reviewable.
  • Implemented a lightweight A/B test comparing creation-button copy and tracking local visit-to-create and create-to-save behavior.

Focus Bakery

Gamified Productivity · Interaction Design · JavaScript

A bakery-themed focus companion where each completed focus round unlocks an ingredient. After five rounds, the user bakes a cute treat and adds it to a growing shop collection. The prototype reframes productivity as a warm progression system instead of a pressure-based task tracker.

Live demo GitHub repo Design process

What I built

  • Designed a clear loop: focus round → ingredient → recipe → baked item → shop collection.
  • Created a scalable recipe system with multiple bakery items, progress states, collection cards, and local persistence.
  • Added warm reflection notes so users can connect completed work to a small moment of progress.

Process

  • Documented wireframes, interaction flow, content decisions, accessibility considerations, and future expansion paths.
  • Used playful interaction patterns to make productivity feel motivating without relying on guilt or pressure.
  • Designed the system so new recipes, seasonal menus, shop decorations, and recap cards can be added later.

Twitter Storage & Retrieval System

Python · MySQL · Redis · SQL

A Twitter-style backend system built to support tweet posting and home timeline retrieval at scale. I designed relational schemas for users, tweets, and follower relationships, implemented API-style functions like postTweet and getHomeTimeline, and benchmarked relational versus in-memory storage tradeoffs.

Technical focus

  • Benchmarked 1M tweet insertions and 50K timeline requests.
  • Improved insertion throughput from approximately 3,906 to 5,574 calls per second using Redis.
  • Evaluated read/write performance tradeoffs for high-volume social media workloads.

MongoDB News Classification Data Pipeline

Python · MongoDB · JSON · Aggregation Pipelines

A document-store data pipeline for transforming raw sarcasm headline data into structured MongoDB-ready documents. I cleaned malformed URLs, extracted source domains, created nested headline records, and wrote aggregation pipelines to analyze sarcasm rates and keyword trends.

Data modeling

  • Designed a flexible document schema with nested records and derived text features.
  • Used mongoimport, indexing, and aggregation stages including $unwind, $group, and $project.
  • Compared modeling tradeoffs for nested retrieval, analytics queries, and scalable ingestion.

MySQL · R · SQL · ETL

A normalized analytics database for aircraft incident data. I designed a 3NF MySQL schema, built an ETL pipeline in R, deployed a cloud-hosted MySQL database with secure SSL, and produced SQL-driven analytics across monthly, airline, and yearly trends.

Analytics system

  • Created modular SQL query patterns for reusable analytics access.
